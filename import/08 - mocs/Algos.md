# 🧠 How to LeetCode — The Pattern Matching Guide

## 👊 TL;DR: It’s All Pattern Matching

LeetCode is not about raw IQ. It’s about **pattern recognition**. Nearly every question fits into a repeatable pattern — whether it’s sliding window, binary search on answer, or backtracking.

Master the **core patterns**, then solve every **tagged problem** with that pattern. Do enough reps and your brain starts auto-completing the solution before you type.

---

## 🔁 Step-by-Step Plan

### Step 1: Learn the Patterns (Don’t Skip This)

Before solving anything, **understand the key patterns**. These are templates you'll use again and again:

🧩 **Go through these curated resources:**

- 📘 [Sean Prashad’s Leetcode Patterns](https://seanprashad.com/leetcode-patterns/)
- 📘 [LeetCode Cheatsheet Explore Card](https://leetcode.com/explore/interview/card/cheatsheets/720/resources/4725/)
- 🎥 [Video: 15 Must-Know Patterns (William Lin)](https://www.youtube.com/watch?v=GPIuPRqDGG8&t=272s)
- 📘 [Blind Blog: Top LeetCode Patterns](https://www.teamblind.com/blog/top-leetcode-patterns-coding-interview-questions/)
- 🎥 [Video: Ace Interviews with Patterns (NeetCode)](https://www.youtube.com/watch?v=0y74ixH2E8Y)

### Step 2: Understand What Each Pattern _Looks_ Like

Create flashcards or a Notion doc summarizing:

- 🔁 When to apply it
- 🧠 The mental model ("What’s changing?")
- 🧪 Template code
- 🧩 1-2 key example problems

Some examples:

- **Sliding Window**: Fixed or dynamic window for substrings, arrays
- **Two Pointers**: Sorted arrays/lists, merging, in-place swaps
- **Binary Search on Answer**: Searching on _range_, not index
- **Backtracking**: Try-all-paths (recursion with undo)
- **Graph Traversals**: DFS/BFS, topological sort
- **Union-Find**: Connected components
- **DP**: Overlapping subproblems → table or memo

---

## ✅ Step 3: Grind Tagged Problems by Pattern

Do every tagged problem under each pattern. You’ll feel the repetition — **that’s the point.** By the 5th version, it clicks.

Start with: 👉 [Sean Prashad’s tagged problems list](https://seanprashad.com/leetcode-patterns/#problems)

And supplement with: 👉 [Blind 75 + Extended](https://leetcode.com/explore/interview/card/cheatsheets/720/resources/4725/)

Log progress. Use:

- ✅ Notion/Excel tracker
- ✅ Leetcode study plans
- ✅ Code notebook w/ explanations

---

## ⚙️ Step 4: Mastery Loop – Refactor & Explain

For each problem:

1. Try it yourself
2. If stuck: recognize the pattern, then write the template
3. After solving:
    - 🪞 Explain the code to yourself out loud
    - 🔁 Refactor it from scratch after a few days
    - 🧠 Think: _What was the key insight or pattern?_

---

## 🧨 Bonus Tips

- 🔍 Filter LeetCode problems by tag (e.g. "two pointers", "heap", "union find")
- 💬 Join Discord/Reddit/Blind communities to discuss edge cases
- ⌛ Set 30–45 min timer per problem. If stuck, **don't brute force** – find the pattern!

---

## 🚀 Summary

> “Interviews are just pattern matching under pressure.”

📚 Learn patterns

🧠 Recognize them in problems

💪 Drill tagged problems

🧘 Reflect and explain your logic

✅ Repeat until it’s second nature

You got this.
